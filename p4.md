### Loading Strategy
- `bge-m3` embedding model and `bge-reranker-v2-m3` are **always loaded in VRAM** (every query uses both)
- Qwen2.5:14b loads on first local agent call; set `OLLAMA_KEEP_ALIVE=60m`
- If VRAM OOM occurs: switch `OLLAMA_KEEP_ALIVE=0` and load/unload per call (adds ~3s per local agent call), or drop to Qwen2.5:7b-Q4_K_M permanently
- Separate CUDA streams for embedding and reranking models to reduce context switching

```python
from sentence_transformers import CrossEncoder

# Expected score range after sigmoid: [0.0, 1.0]
# Scores > 0.5 indicate positive relevance. All reranker_threshold values
# in RETRIEVAL_CONFIGS (0.25–0.8) and Quality Assessor heuristics assume
# this normalized range. Do NOT remove the sigmoid.

# Embedding model — no special activation needed
from sentence_transformers import SentenceTransformer
embedding_model = SentenceTransformer(
```

### CRITICAL: All Embedding and Reranking Calls Must Be Non-Blocking

`SentenceTransformer.encode()` and cross-encoder `predict()` are **synchronous** CPU/GPU operations. Calling them directly inside `async` functions blocks the entire asyncio event loop — every other coroutine is frozen until encoding completes. This is not optional to fix; it breaks concurrent request handling.

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

# Module-level executors — separate pools prevent embed/rerank from starving each other.
# With a single executor of max_workers=2, one request uses both threads (embed + rerank),
# blocking any concurrent request entirely. Two dedicated executors fix this.
_embed_executor = ThreadPoolExecutor(max_workers=2, thread_name_prefix="embed")
_rerank_executor = ThreadPoolExecutor(max_workers=2, thread_name_prefix="rerank")

async def embed_texts_async(texts: list[str]) -> "np.ndarray":
    """
    Non-blocking wrapper for synchronous SentenceTransformer.encode().
    Always use this in async context — never call model.encode() directly.
    
    Returns np.ndarray (not list[list[float]]). Callers must call .tolist()
    when passing to Qdrant or other APIs that expect plain Python lists.
    """
    loop = asyncio.get_running_loop()  # ← NOT get_event_loop() — deprecated in 3.10+, broken in 3.12
    return await loop.run_in_executor(
        _embed_executor,
        lambda: embedding_model.encode(
            texts,
            normalize_embeddings=True,
            batch_size=32,
            show_progress_bar=False,
        )
    )

async def rerank_async(query: str, passages: list[str]) -> "np.ndarray":
    """
    Non-blocking wrapper for synchronous cross-encoder reranking.
    
    Returns np.ndarray (not list[float]). CrossEncoder.predict() returns
    numpy array. Callers must convert individual scores: float(score).
    """
    loop = asyncio.get_running_loop()  # ← NOT get_event_loop()
    pairs = [[query, p] for p in passages]
    return await loop.run_in_executor(
        _rerank_executor,
        lambda: reranker_model.predict(pairs)
    )
```

Apply the same `run_in_executor` pattern everywhere a synchronous model call appears in an async context — ingestion pipeline, retrieval executor, and reranking.

### Rate Limits
```

```

### RPM Rate Limiting — Sliding Window (Required Under Concurrent Load)

At 15 RPM for `gemini-3.1-flash-lite`, a single query uses 3 calls in ~2 seconds. Two concurrent users immediately hit the ceiling. `BudgetTracker` must enforce both **daily RPD limits** and **per-minute RPM limits** with a sliding window, not just count totals.

```python
import asyncio
import time
from collections import deque

class RateLimiter:
    """
    Sliding window rate limiter per model.
    Thread-safe for concurrent async requests.
    
    IMPORTANT: The lock is released BEFORE sleeping so other coroutines
    can proceed. After waking, the lock is re-acquired and timestamps
    are re-purged to handle changes made while sleeping.
    """
    def __init__(self, rpm_limit: int):
        self.rpm_limit = rpm_limit
        self._call_timestamps: deque[float] = deque()
        self._lock = asyncio.Lock()

    async def acquire(self) -> None:
        """Wait until a call slot is available within the current 60-second window."""
        while True:
            async with self._lock:
                now = time.monotonic()
                # Purge timestamps older than 60 seconds
                while self._call_timestamps and self._call_timestamps[0] < now - 60:
                    self._call_timestamps.popleft()

                if len(self._call_timestamps) < self.rpm_limit:
                    # Slot available — record and return
                    self._call_timestamps.append(time.monotonic())
                    return

                # No slot available — calculate wait time
                wait_time = 60.0 - (now - self._call_timestamps[0])

            # CRITICAL: Lock is RELEASED here before sleeping.
            # Without this, asyncio.sleep() holds the lock for up to 60s,
            # serializing ALL concurrent requests behind the sleeping coroutine.
            if wait_time > 0:
                await asyncio.sleep(wait_time)
            # Loop back: re-acquire lock, re-purge timestamps, re-check slot
```

### BudgetTracker — Full Specification

The `BudgetTracker` must be:
- **Persistent across restarts** — stored in Postgres, not in-memory
- **Per-model** — tracks `synthesis_primary` and `agent_workhorse` separately
- **Thread-safe** — uses DB transactions for increment
- **UTC-midnight resetting** — checked on every call
- **Exposed in the UI** — Streamlit sidebar shows live budget status

```python
# src/llm/budget_tracker.py

from dataclasses import dataclass
from datetime import datetime, timezone
import asyncpg

@dataclass
class ModelBudget:
    model_key: str
    daily_limit: int
    used_today: int
    reset_date: str   # ISO date string UTC

class BudgetTracker:
    """
    Singleton that persists daily API call counts to Postgres.
    Call get_model_for_agent() before every LiteLLM call to get
    the appropriate model string given current budget state.

    Schema (auto-created on startup):
        CREATE TABLE IF NOT EXISTS api_budget (
            model_key TEXT PRIMARY KEY,
            used_today INTEGER DEFAULT 0,
            reset_date DATE DEFAULT CURRENT_DATE
        );
    
    INITIALIZATION:
        # BudgetTracker uses asyncpg.create_pool() which is async — it CANNOT
        # be called in __init__. Use the async factory classmethod:
        tracker = await BudgetTracker.get_instance(postgres_url)
    
    RATE LIMITERS:
        # asyncio.Lock() requires a running event loop. Class-level attributes
        # are evaluated at import time — BEFORE any event loop is running.
        # In Python 3.10+, asyncio.Lock() outside a running loop raises
        # DeprecationWarning or RuntimeError. Rate limiters are created lazily
        # on first use via _get_rate_limiter().
    """
    _instance = None
    
    DAILY_LIMITS = {
        "synthesis_primary": 18,    # gemini-3.5-flash: leave 2 RPD buffer from 20
        "agent_workhorse": 480,     # gemini-3.1-flash-lite: leave 20 RPD buffer from 500
    }
    
    RPM_LIMITS = {
        "synthesis_primary": 5,
        "agent_workhorse": 15,
    }
    
    # DO NOT instantiate RateLimiter at class definition time.
    # asyncio.Lock() inside RateLimiter.__init__ will fail before event loop starts.
    _rate_limiters: dict = {}  # populated lazily on first use
    _instance_lock: asyncio.Lock | None = None  # lazily created — see get_instance()
    
    @classmethod
    async def get_instance(cls, postgres_url: str) -> "BudgetTracker":
        """
        Async factory — the ONLY way to create a BudgetTracker.
        asyncpg.create_pool() is async and cannot be called in __init__.
        
        Uses double-checked locking to prevent race condition where two
        concurrent startup requests both see _instance is None and create
        duplicate connection pools (the second silently orphans the first).
        
        Args:
            postgres_url: PostgreSQL connection string.
        Returns:
            Singleton BudgetTracker instance with initialized connection pool.
        """
        # Lazy lock creation — safe because this is called inside a running event loop
        if cls._instance_lock is None:
            cls._instance_lock = asyncio.Lock()
        async with cls._instance_lock:
            if cls._instance is None:
                instance = cls.__new__(cls)
                instance._pool = await asyncpg.create_pool(
                    postgres_url, min_size=2, max_size=5
                )
                await instance._ensure_schema()
                cls._instance = instance
        return cls._instance
    
    async def _ensure_schema(self) -> None:
        """Creates the api_budget table if it doesn't exist."""
        async with self._pool.acquire() as conn:
            await conn.execute("""
                CREATE TABLE IF NOT EXISTS api_budget (
                    model_key TEXT PRIMARY KEY,
                    used_today INTEGER DEFAULT 0,
                    reset_date DATE DEFAULT CURRENT_DATE
                )
            """)
            # Ensure rows exist for each model
            for model_key in self.DAILY_LIMITS:
                await conn.execute("""
                    INSERT INTO api_budget (model_key, used_today, reset_date)
                    VALUES ($1, 0, CURRENT_DATE)
                    ON CONFLICT (model_key) DO NOTHING
                """, model_key)
    
    def _get_rate_limiter(self, model_key: str) -> "RateLimiter":
        """
        Lazily creates RateLimiter instances on first use.
        This ensures asyncio.Lock() is called inside a running event loop,
        not at class definition / import time.
        """
        if model_key not in self._rate_limiters:
            self._rate_limiters[model_key] = RateLimiter(
                rpm_limit=self.RPM_LIMITS[model_key]
            )
        return self._rate_limiters[model_key]
    
    async def get_model_for_synthesis(self) -> str:
        """
        Returns gemini-3.5-flash model string if daily budget remains,
        otherwise falls back to gemini-3.1-flash-lite.
        ⚠ Verify these strings match LiteLLM provider docs before use.
        
        CRITICAL: Acquire the rate limiter slot BEFORE consuming the daily budget.
        If the task is cancelled while sleeping in acquire() (e.g., timeout, client
        disconnect), the budget is NOT debited for a call that was never sent.
        The atomic conditional UPDATE in _try_consume() prevents the TOCTOU race.
        """
        # Read-only check to avoid rate-limiter waiting if budget is already exhausted
        if not await self._budget_available("synthesis_primary"):
            return await self.get_model_for_agent()

        await self._get_rate_limiter("synthesis_primary").acquire()
        if await self._try_consume("synthesis_primary"):
            return "gemini/gemini-3.5-flash"   # ⚠ VERIFY STRING
        return await self.get_model_for_agent()

    async def get_model_for_agent(self) -> str:
        """
        Returns gemini-3.1-flash-lite if budget remains, else local Ollama fallback.
        ⚠ Verify model string against LiteLLM docs before use.
        
        CRITICAL: Acquire the rate limiter slot BEFORE consuming the daily budget.
        If the task is cancelled while sleeping in acquire() (e.g., timeout, client
        disconnect), the budget is NOT debited for a call that was never sent.
        """
        # Read-only check to avoid rate-limiter waiting if budget is already exhausted
        if not await self._budget_available("agent_workhorse"):
            return "ollama/qwen2.5:14b-instruct-q4_k_m"

        await self._get_rate_limiter("agent_workhorse").acquire()
        if await self._try_consume("agent_workhorse"):
            return "gemini/gemini-3.1-flash-lite"  # ⚠ VERIFY STRING
        return "ollama/qwen2.5:14b-instruct-q4_k_m"

    async def get_budget_status(self) -> dict:
        """For Streamlit sidebar display."""
        status = {}
        for model_key in self.DAILY_LIMITS:
            budget = await self._load_budget(model_key)
            status[model_key] = {
                "used": budget.used_today,
                "limit": self.DAILY_LIMITS[model_key],
                "remaining": self.DAILY_LIMITS[model_key] - budget.used_today,
                "reset_date": budget.reset_date,
            }
        return status

    async def _budget_available(self, model_key: str) -> bool:
        """Read-only check for UI display. Do NOT use for consumption decisions — use _try_consume() instead."""
        budget = await self._load_budget(model_key)
        today = datetime.now(timezone.utc).date().isoformat()
        if budget.reset_date != today:
            return True
        return budget.used_today < self.DAILY_LIMITS[model_key]

    async def _try_consume(self, model_key: str) -> bool:
        """
        Atomic check-and-increment: returns True and increments counter if budget
        remains, returns False without incrementing if exhausted.
        
        CRITICAL: This replaces the old _budget_available() + _increment() pattern
        which had a TOCTOU race: between checking and incrementing, a concurrent
        request could pass the same check, causing both to increment and overshoot
        the daily limit (e.g., 21/20 RPD). The single conditional UPDATE below
        is atomic at the DB level — the WHERE clause ensures the increment only
        fires if used_today < limit at execution time.
        """
        today = datetime.now(timezone.utc).date().isoformat()
        async with self._pool.acquire() as conn:
            # Reset if new day (idempotent, same conditional as _reset)
            await conn.execute(
                "UPDATE api_budget SET used_today = 0, reset_date = $1::date "
                "WHERE model_key = $2 AND reset_date < $1::date",
                today, model_key
            )
            # Atomic conditional increment — prevents TOCTOU race
            result = await conn.execute(
                "UPDATE api_budget SET used_today = used_today + 1 "
                "WHERE model_key = $1 AND used_today < $2",
                model_key, self.DAILY_LIMITS[model_key]
            )
            # result is "UPDATE N" where N is rows affected
            return result == "UPDATE 1"

    async def _load_budget(self, model_key: str) -> ModelBudget:
        """
        Loads current budget state from Postgres.
        Returns ModelBudget dataclass with used_today and reset_date.
        """
        async with self._pool.acquire() as conn:
            row = await conn.fetchrow(
                "SELECT model_key, used_today, reset_date FROM api_budget WHERE model_key = $1",
                model_key
            )
            if row is None:
                # First use — insert default
                await conn.execute(
                    "INSERT INTO api_budget (model_key, used_today, reset_date) VALUES ($1, 0, CURRENT_DATE)",
                    model_key
                )
                return ModelBudget(
                    model_key=model_key, daily_limit=self.DAILY_LIMITS[model_key],
                    used_today=0, reset_date=datetime.now(timezone.utc).date().isoformat()
                )
            return ModelBudget(
                model_key=row["model_key"],
                daily_limit=self.DAILY_LIMITS[model_key],
                used_today=row["used_today"],
                reset_date=row["reset_date"].isoformat() if hasattr(row["reset_date"], "isoformat") else str(row["reset_date"]),
            )
```

### JSON Mode — Required for All Structured-Output Agents

Agents 1, 5, 6, 7, and 8 expect structured JSON output. Without enforcing JSON mode, Gemini intermittently returns markdown-wrapped JSON or prose, requiring fragile regex extraction fallbacks.

```python
# src/llm/litellm_wrapper.py

async def call_structured_agent(
    system_prompt: str,
    user_prompt: str,
    model: str,
    temperature: float = 0.0,
    max_tokens: int = 1000,
) -> dict:
    """
    Wrapper for all agents that return JSON.
    Enforces JSON mode via response_format parameter.
    Includes retry on JSON parse failure (max 2 retries).
    """
    for attempt in range(3):
        response = await litellm.acompletion(
            model=model,
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": user_prompt},
            ],
            temperature=temperature,
            max_tokens=max_tokens,
            response_format={"type": "json_object"},  # Enforces JSON mode
        )
        raw = response.choices[0].message.content
        try:
            # Strip accidental markdown fences before parsing
            clean = raw.strip().removeprefix("```json").removesuffix("```").strip()
            return json.loads(clean)
        except json.JSONDecodeError as e:
            if attempt == 2:
                raise ValueError(f"Agent returned invalid JSON after 3 attempts: {e}\nRaw: {raw}")
            continue
```

```python
from qdrant_client.models import (

            ),

    Creates the parent-chunk collection for context expansion.
    Parent text blobs are large — do NOT set always_ram on payload.
    At 50k chunks × ~8KB parent text = ~400MB per deal; forcing this
    into RAM exhausts the 32GB system RAM budget at scale.
    Vectors are on disk; payload (containing parent_text) goes to disk.

    INGESTION COMPLIANCE NOTE:
    The ingestion pipeline MUST write the `contains_pii` (0 or 1) and
    `is_current_version` (0 or 1) metadata fields to both the main child-chunk

        collection_name=PARENT_COLLECTION_NAME,

                distance=Distance.COSINE,

        ),
```

### Core Principle: Metadata as Filter, Not Third Channel

### Deduplication — Full Specification

```python
from qdrant_client.models import ScoredPoint

def flatten_deduplicate(
    result_lists: list[list[ScoredPoint]],
) -> list[ScoredPoint]:
    """
    Merges multiple ranked result lists into one deduplicated list.
    When the same chunk_id appears in multiple lists (e.g., from query expansions),
    keeps the instance with the highest individual score.
    Returns sorted by score descending.

    NOTE: This is the one place raw retrieval scores (cosine similarity for dense,
    BM25 for sparse) drive a ranking decision, which is slightly in tension with
    the "pure rank-based, no normalization" RRF principle stated elsewhere.
    This is acceptable because: (1) cosine scores ARE comparable across queries
    against the same corpus, and (2) this deduplication happens BEFORE RRF, so
    the raw-score comparison only determines which duplicate to keep, not the
    final ranking. RRF itself still uses ranks only.

    Used to merge results from: original query + N query expansions,
    before feeding the combined pool into RRF fusion.

    Args:
        result_lists: Multiple ranked lists from parallel searches.
    Returns:
        Single deduplicated list sorted by score descending.
    """
    seen: dict[str, ScoredPoint] = {}
    for result_list in result_lists:
        for point in result_list:
            chunk_id = point.payload["chunk_id"]
            if chunk_id not in seen or point.score > seen[chunk_id].score:
                seen[chunk_id] = point
    return sorted(seen.values(), key=lambda p: p.score, reverse=True)
```

### Pure RRF Fusion (Rank-Based Only — No Score Normalization)

```python
def reciprocal_rank_fusion(
    dense_results: list[ScoredPoint],
    sparse_results: list[ScoredPoint],
    k: int = 60,
    dense_weight: float = 0.6,
    sparse_weight: float = 0.4,
) -> list[tuple[str, float]]:
    """
    Pure rank-based Reciprocal Rank Fusion.
    Scores from dense and sparse search are IGNORED — only rank positions matter.
    This makes RRF inherently scale-agnostic: cosine similarity (bounded [-1,1])
    and BM25 scores (unbounded) cannot distort each other because neither is used.

    The formula: rrf_score(d) = Σ weight_i / (k + rank_i(d))
    where k=60 prevents very high-ranked items from dominating absolutely.

    Do NOT add score normalization before this function — it converts RRF
    into a different algorithm and reintroduces the scale problem.

    Args:
        dense_results: Ranked list from dense vector search (already deduplicated).
        sparse_results: Ranked list from BM25 sparse search (already deduplicated).
        k: Smoothing constant. 60 is standard; increase to reduce top-rank dominance.
        dense_weight: Contribution weight for dense channel.
        sparse_weight: Contribution weight for sparse channel.
    Returns:
        List of (chunk_id, rrf_score) sorted by score descending.
    Raises:
        ValueError: If both result lists are empty.
    """
    if not dense_results and not sparse_results:
        raise ValueError("Both dense and sparse results are empty — cannot perform RRF")

    scores: dict[str, float] = {}

    for rank, point in enumerate(dense_results, start=1):
        chunk_id = point.payload["chunk_id"]
        scores[chunk_id] = scores.get(chunk_id, 0.0) + dense_weight / (k + rank)

    for rank, point in enumerate(sparse_results, start=1):
        chunk_id = point.payload["chunk_id"]
        scores[chunk_id] = scores.get(chunk_id, 0.0) + sparse_weight / (k + rank)

    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

```python
    The same applies to table_id in fetch_table_siblings and parent_chunk_id
    in expand_context — all IDs embed the deal_id prefix.

    Args:
        chunk_ids: List of chunk_id strings to retrieve.
        client: AsyncQdrantClient instance. If None, uses get_qdrant_client().
    Returns:
        List of payload dicts in the same order as input chunk_ids.
        Missing IDs are silently skipped (logged as warnings).
    Raises:
        QdrantException: On connection failure after retries.
    """
    if client is None:
        client = get_qdrant_client()
    
    results = await client.scroll(
        collection_name=COLLECTION_NAME,
        scroll_filter=Filter(must=[
            FieldCondition(key="chunk_id", match=MatchAny(any=chunk_ids))
        ]),
        limit=len(chunk_ids),
        with_payload=True,
        with_vectors=False,
    )
    # Return in original rank order — preserves RRF ranking
    payload_map = {p.payload["chunk_id"]: p.payload for p in results[0]}
    ordered = [payload_map[cid] for cid in chunk_ids if cid in payload_map]
    
    if len(ordered) < len(chunk_ids):
        missing = set(chunk_ids) - set(payload_map.keys())
        logger.warning(f"fetch_chunks_by_ids: {len(missing)} IDs not found: {missing}")
    
    return ordered
```

### Full Hybrid Search Function

```python
from qdrant_client.models import NamedVector, NamedSparseVector, Filter, FieldCondition, MatchValue
from src.vector_db.constants import QUANTIZED_SEARCH_PARAMS  # ← REQUIRED: defined in constants.py

async def hybrid_search(
    query_text: str,
    query_vector: list[float],
    query_sparse: SparseVector,
    deal_id: str,
    metadata_filters: dict,
    top_k_dense: int = 40,
    top_k_sparse: int = 40,
    client: AsyncQdrantClient = None,
) -> tuple[list[ScoredPoint], list[ScoredPoint]]:
    """
    Executes dense and sparse searches in parallel with metadata constraints
    applied as Qdrant filter parameters — NOT as a separate retrieval channel.

    Args:
        query_text: Original query string (used for logging).
        query_vector: Dense embedding of the query.
        query_sparse: BM25 sparse vector of the query.
        deal_id: Mandatory deal isolation filter.
        metadata_filters: Dict of payload fields to filter on
                          (e.g., {"fiscal_year": "FY2023", "is_current_version": 1}).
        top_k_dense: Number of candidates from dense search.
        top_k_sparse: Number of candidates from sparse search.
        client: AsyncQdrantClient instance.
    Returns:
        Tuple of (dense_results, sparse_results) as ScoredPoint lists.
    """
    qdrant_filter = _build_filter(deal_id, metadata_filters)

    dense_task = client.search(
        collection_name=COLLECTION_NAME,
        query_vector=NamedVector(name="dense", vector=query_vector),
        query_filter=qdrant_filter,
        limit=top_k_dense,
        search_params=QUANTIZED_SEARCH_PARAMS,  # rescore=True, oversampling=2.0
        with_payload=True,
    )
    sparse_task = client.search(
        collection_name=COLLECTION_NAME,
        query_vector=NamedSparseVector(name="sparse", vector=query_sparse),
        query_filter=qdrant_filter,
        limit=top_k_sparse,
        with_payload=True,
    )

    dense_results, sparse_results = await asyncio.gather(dense_task, sparse_task)
    return dense_results, sparse_results


def _build_filter(deal_id: str, metadata_filters: dict) -> Filter:
    """
    Builds a Qdrant Filter from deal_id (always applied) and optional metadata.
    deal_id is mandatory — never execute a search without it.
    
    Default exclusions applied automatically:
    - is_current_version=1 (unless explicitly overridden in metadata_filters)
    - contains_pii=0 (PII-flagged content excluded by default per compliance policy;
      pass include_pii=True in metadata_filters to override for authorized users)
    """
    conditions = [FieldCondition(key="deal_id", match=MatchValue(value=deal_id))]
    # Always filter to current versions unless explicitly overridden
    if "is_current_version" not in metadata_filters:
        conditions.append(FieldCondition(key="is_current_version", match=MatchValue(value=1)))
    # CRITICAL: Exclude PII-flagged content by default.
    # The project spec states "PII detection at ingestion — flags HR/comp documents,
    # excludes from retrieval by default." Without this filter, PII content is
    # retrievable by any query, violating the stated compliance policy.
    #
    # NOTE: Use .get() on a shallow copy — NOT .pop() on the original dict.
    # _build_filter() is called once per query expansion via asyncio.gather().
    # .pop() on the shared dict would consume include_pii on the first call,
    # causing all subsequent concurrent calls to silently fall back to PII-excluded
    # even when an authorized analyst explicitly set include_pii=True.
    local_filters = dict(metadata_filters)  # shallow copy — never mutate caller's dict
    include_pii = local_filters.pop("include_pii", False)
    if not include_pii:
        conditions.append(FieldCondition(key="contains_pii", match=MatchValue(value=0)))
    for key, value in local_filters.items():
        if value is not None:
            conditions.append(FieldCondition(key=key, match=MatchValue(value=value)))
    return Filter(must=conditions)
```

```python
# src/data_processing/excel_normalizer.py
import re
from dataclasses import dataclass

SCALE_PATTERNS = {
    r'in\s+thousands?|\$000s?|×\s*1[,.]?000(?!\s*,?000)': 1_000.0,
    r'in\s+millions?|\$M\b|MM\b|×\s*1[,.]?000[,.]?000':    1_000_000.0,
    r'in\s+billions?|\$B\b|×\s*1[,.]?000[,.]?000[,.]?000': 1_000_000_000.0,
}

@dataclass
class TableNormalizationMeta:
    currency: str           # "USD", "EUR", "GBP", etc.
    scale_factor: float     # 1.0, 1_000.0, 1_000_000.0, 1_000_000_000.0
    scale_label: str        # "units", "thousands", "millions", "billions"

class ExcelNormalizer:
    """
    Detects unit scale and currency from column/row headers before value extraction.
    
    CRITICAL: Without normalization, cross-document comparison produces false
    inconsistency alarms. A value of 45,200 (scale=thousands) and 45.2 (scale=millions)
    are identical when normalized — the Financial Verifier must compare normalized
    values only, never raw values across documents.

    All Financial Verification Agent comparisons MUST use normalized_value.
    """
    def detect_scale(self, header_cells: list[str]) -> TableNormalizationMeta:
        """
        Scans header cells for scale indicators.
        Returns TableNormalizationMeta with scale_factor and scale_label.
        Defaults to scale_factor=1.0 if no indicator found.
        """
        combined = " ".join(str(c) for c in header_cells).lower()
        for pattern, factor in SCALE_PATTERNS.items():
            if re.search(pattern, combined, re.IGNORECASE):
                label = {1_000.0: "thousands", 1_000_000.0: "millions",
                         1_000_000_000.0: "billions"}.get(factor, "units")
                return TableNormalizationMeta(
                    currency=self._detect_currency(combined),
                    scale_factor=factor,
                    scale_label=label,
                )
        return TableNormalizationMeta(currency=self._detect_currency(combined),
                                     scale_factor=1.0, scale_label="units")

    def _detect_currency(self, text: str) -> str:
        """Detects currency from header text. Returns ISO 4217 code or 'UNKNOWN'."""
        if "$" in text or "usd" in text.lower(): return "USD"
        if "€" in text or "eur" in text.lower(): return "EUR"
        if "£" in text or "gbp" in text.lower(): return "GBP"
        if "cad" in text.lower() or "c$" in text: return "CAD"
        if "aud" in text.lower() or "a$" in text: return "AUD"
        if "¥" in text or "jpy" in text.lower(): return "JPY"
        if "chf" in text.lower(): return "CHF"
        if "₹" in text or "inr" in text.lower(): return "INR"
        if "cny" in text.lower() or "rmb" in text.lower() or "元" in text: return "CNY"
        # Return UNKNOWN instead of silently assuming USD.
        # The Financial Verifier will flag UNKNOWN currencies for analyst review
        # rather than producing silent misclassification in cross-border deals.
        return "UNKNOWN"
```

```python
import pandas as pd
from typing import Literal

RepresentationType = Literal["narrative", "row_by_row", "metrics_summary", "markdown"]

class FinancialTableConverter:
    """
    Generates 4 chunk representations per financial table, all sharing a table_id.
    
    CRITICAL RULE: All derived metrics (margins, CAGR, YoY growth) are computed
    DETERMINISTICALLY in pandas — NEVER via LLM. LLMs get arithmetic wrong silently.
    
    Every computed value carries a citation_chain showing its inputs:
    content_type="computed_metric" distinguishes these from verbatim source numbers.
    The synthesis and validation agents must treat computed_metric chunks differently:
    they cannot be verified against source text, only against their citation_chain.
    """
    def generate_all_representations(
        self,
        df: pd.DataFrame,
        meta: TableNormalizationMeta,
        table_id: str,
        source_metadata: dict,
    ) -> list[dict]:
        return [
            self._narrative(df, meta, table_id, source_metadata),
            self._row_by_row(df, meta, table_id, source_metadata),
            self._metrics_summary(df, meta, table_id, source_metadata),  # pandas only
            self._markdown(df, meta, table_id, source_metadata),
        ]

    def _metrics_summary(self, df, meta, table_id, source_meta) -> dict:
        """Computes CAGR, margins, YoY growth via pandas. Each metric gets citation_chain."""
        metrics = {}
        if "Revenue" in df.index:
            revenue_row = df.loc["Revenue"] * meta.scale_factor
            years = sorted([c for c in revenue_row.index if str(c).startswith("FY")])
            if len(years) >= 2:
                n = len(years) - 1
                cagr = (revenue_row[years[-1]] / revenue_row[years[0]]) ** (1/n) - 1
                metrics["Revenue CAGR"] = {
                    "value": round(cagr * 100, 2),
                    "unit": "%",
                    "citation_chain": f"CAGR(Revenue[{years[0]}={revenue_row[years[0]]:,.0f} → {years[-1]}={revenue_row[years[-1]]:,.0f}], n={n})",
                    "content_type": "computed_metric",
                }
        # ... similar for margins, YoY, etc.
        return {"table_representation": "metrics_summary", "metrics": metrics,
                "table_id": table_id, "content_type": "computed_metric", **source_meta}
```

```python
# src/workflow/orchestrator.py
# CRITICAL: The rewrite loop connects query_rewriter → parallel_retrieval_executor
# NOT query_rewriter → retrieval_strategy.
# retrieval_strategy is now deterministic code called AT THE START of executor,
# not a separate node. The rewriter emits updated_retrieval_config directly into
# state, and the executor reads it from state — no routing through retrieval_strategy.

from langgraph.graph import StateGraph, END
from langgraph.checkpoint.postgres import PostgresSaver

def build_graph(checkpointer: PostgresSaver) -> ...:
    workflow = StateGraph(AgentState)

    # Add nodes
    workflow.add_node("query_intelligence",        query_intelligence_node)
    workflow.add_node("parallel_retrieval_executor", retrieval_executor_node)
    # retrieval_strategy is NOT a node — it's deterministic code called inside executor
    workflow.add_node("financial_verifier",        financial_verifier_node)
    workflow.add_node("quality_assessor",          quality_assessor_node)
    workflow.add_node("query_rewriter",            query_rewriter_node)
    workflow.add_node("answer_synthesizer",        answer_synthesizer_node)
    workflow.add_node("hallucination_validator",   hallucination_validator_node)

    # Entry point
    workflow.set_entry_point("query_intelligence")

    # Linear flow
    workflow.add_edge("query_intelligence", "parallel_retrieval_executor")

    # Conditional: run financial verifier only if needed
    workflow.add_conditional_edges(
        "parallel_retrieval_executor",
        route_to_financial_verifier,
        {
            "financial_verifier": "financial_verifier",
            "quality_assessor": "quality_assessor",
        }
    )
    workflow.add_edge("financial_verifier", "quality_assessor")

    # Quality gate with rewrite loop
    workflow.add_conditional_edges(
        "quality_assessor",
        route_after_quality_check,
        {
            "answer_synthesizer": "answer_synthesizer",
            "query_rewriter": "query_rewriter",
            "insufficient_context": "answer_synthesizer",  # forced refusal path
        }
    )

    # ← CRITICAL EDGE: rewriter goes to executor, NOT to a retrieval_strategy node
    workflow.add_edge("query_rewriter", "parallel_retrieval_executor")

    # Answer pipeline
    workflow.add_edge("answer_synthesizer", "hallucination_validator")
    workflow.add_conditional_edges(
        "hallucination_validator",
        route_after_validation,
        {
            "end": END,
            "retry_synthesis": "answer_synthesizer",
        }
    )

    return workflow.compile(checkpointer=checkpointer)

# IMPORTANT: All nodes use `async def`. When invoking the compiled graph,
# use `await app.ainvoke(state, config=config)` — NOT `app.invoke()`.
# `app.invoke()` is for synchronous nodes only and will deadlock if any
# node is async.
#
# PostgresSaver thread_id construction:
# The checkpointer keys state to (deal_id, session_id). Build the config as:
#   config = {"configurable": {"thread_id": f"{deal_id}:{session_id}"}}
# This composite key ensures:
#   - Different deals never share checkpoint state (confidentiality)
#   - Different sessions within the same deal have independent rewrite histories
#   - Checkpoint resume works correctly across server restarts
#
# Example invocation:
#   app = build_graph(checkpointer)
#   config = {"configurable": {"thread_id": f"{deal_id}:{session_id}"}}
#   result = await app.ainvoke(initial_state, config=config)
```

### Conditional Edge Functions

```python
def route_to_financial_verifier(state: AgentState) -> str:
    if (state["query_type"] == "financial" or
            state["parsed_intent"].get("requires_numerical_precision", False)):
        return "financial_verifier"
    return "quality_assessor"

def _meets_type_threshold(state: AgentState) -> bool:
    """
    Returns True if quality_breakdown meets ALL per-dimension thresholds
    for the current query type. Called by route_after_quality_check.
    
    Uses the THRESHOLDS dict from Agent 5 (Quality Assessor):
        THRESHOLDS = {
            "financial":   {"relevance": 0.7, "completeness": 0.7, "precision": 0.8},
            "legal":       {"relevance": 0.8, "completeness": 0.6, "precision": 0.7},
            ...
        }
    
    Args:
        state: Current AgentState with quality_breakdown and query_type.
    Returns:
        True if every dimension score >= its threshold for this query type.
    """
    breakdown = state.get("quality_breakdown", {})
    thresholds = THRESHOLDS.get(state["query_type"], {})
    return all(breakdown.get(k, 0.0) >= v for k, v in thresholds.items())

def route_after_quality_check(state: AgentState) -> str:
    """Conditional edge function — reads state only, never mutates it."""
    score = state["context_quality_score"]
    iteration = state["rewrite_iteration"]
    if score >= 0.3 and _meets_type_threshold(state):
        return "answer_synthesizer"
    if iteration >= 2:  # Max 2 rewrites (not 3 — reduces worst-case latency)
        # force_refusal is already set by quality_assessor_node (see Agent 5).
        # Edge functions must NOT mutate state — LangGraph's reducer/checkpointing
        # model merges state at superstep boundaries via node returns.
        # Mutating state here bypasses reducers and can break checkpoint resume.
        return "insufficient_context"
    return "query_rewriter"

def route_after_validation(state: AgentState) -> str:
    if (state["validation_status"] == "failed" and
            state.get("validation_attempt", 0) < 1):
        return "retry_synthesis"
    return "end"
```

### Agent 3 — Parallel Retrieval Executor
**No LLM | All embedding/reranking calls via `run_in_executor` (non-blocking)**

```python
async def retrieval_executor_node(state: AgentState) -> AgentState:
    config = state.get("retrieval_config") or get_retrieval_config(
        state["query_type"], state["parsed_intent"]
    )
    queries = [state["current_query"]] + state["parsed_intent"].get("query_expansions", [])

    # Step 1: Embed all queries concurrently (non-blocking via run_in_executor)
    embedding_tasks = [embed_texts_async([q]) for q in queries]
    all_embeddings = await asyncio.gather(*embedding_tasks)
    # IMPORTANT: compute_sparse_bm25 is synchronous CPU computation.
    # With 4 query expansions this is 4 synchronous calls blocking the event loop.
    # Wrap in executor for consistency with embed_texts_async.
    loop = asyncio.get_running_loop()  # ← NOT get_event_loop() — deprecated in 3.10+
    sparse_vectors = await asyncio.gather(*[
        loop.run_in_executor(_embed_executor, compute_sparse_bm25, q)
        for q in queries
    ])

    # Step 2: Run all searches in parallel
    search_tasks = []
    for emb, sparse in zip(all_embeddings, sparse_vectors):
        search_tasks.append(hybrid_search(
            query_text=state["current_query"],
            query_vector=emb[0].tolist(),
            query_sparse=sparse,
            deal_id=state["deal_id"],
            metadata_filters=state["extracted_filters"],
            top_k_dense=config["top_k_dense"],
            top_k_sparse=config["top_k_sparse"],
            client=get_qdrant_client(),
        ))
    all_results = await asyncio.gather(*search_tasks)

    # Step 3: Flatten, deduplicate, RRF
    all_dense = flatten_deduplicate([r[0] for r in all_results])
    all_sparse = flatten_deduplicate([r[1] for r in all_results])
    fused = reciprocal_rank_fusion(all_dense, all_sparse,
                                   dense_weight=config["dense_weight"],
                                   sparse_weight=config["sparse_weight"])

    # Step 4: Fetch chunk payloads for top candidates, rerank (non-blocking)
    top_ids = [chunk_id for chunk_id, _ in fused[:config["reranker_top_k"]]]
    top_chunks = await fetch_chunks_by_ids(top_ids)
    reranked = await rerank_async(state["current_query"],
                                  [c["text"] for c in top_chunks])

    # Sort by reranker score, apply threshold
    scored = sorted(zip(top_chunks, reranked), key=lambda x: x[1], reverse=True)
    filtered = [(c, s) for c, s in scored if s >= config["reranker_threshold"]]
    # CRITICAL: Inject reranker_score into each chunk dict.
    # Without this, the Quality Assessor's heuristic reads c["reranker_score"]
    # and gets a KeyError — the score `s` is discarded by list comprehension otherwise.
    final_chunks = []
    for chunk, score in filtered[:config["final_top_k"]]:
        chunk["reranker_score"] = float(score)
        final_chunks.append(chunk)

    # Step 5: Parent + sibling expansion (pass include_pii from state to enforce compliance)
    include_pii = state.get("extracted_filters", {}).get("include_pii", False)
    expanded = await expand_context(
        final_chunks,
        include_parents=config["use_parent_expansion"],
        include_siblings=config["use_sibling_expansion"],
        include_pii=include_pii,
    )

    return {**state, "retrieval_config": config, "reranked_results": final_chunks,
            "expanded_context": expanded}
```

```python
# src/vector_db/sibling_retrieval.py

async def fetch_table_siblings(
    chunks: list[dict],
    required_representations: list[str],
    client: AsyncQdrantClient,
    include_pii: bool = False,
) -> list[dict]:
    """
    For any chunk with is_table=1, fetches all sibling representations
    sharing the same table_id. This is required because the 4-representation
    design indexes narrative/row_by_row/metrics_summary/markdown separately.
    
    Without this step, retrieval might return only the narrative chunk
    while the answer synthesizer needs the markdown for exact number verification.
    Uses the table_id payload index for efficient filtering.

    Args:
        chunks: Reranked chunks from retrieval (may include non-table chunks).
        required_representations: Which representations to fetch (default: all 4).
        client: AsyncQdrantClient.
        include_pii: Compliance authorization override. If False, filters out PII content.
    Returns:
        Input chunks with table chunks expanded to include all sibling representations.
    """
    table_ids = {c["table_id"] for c in chunks if c.get("is_table") == 1 and c.get("table_id")}
    if not table_ids:
        return chunks

    # NOTE: scroll is acceptable here because we are doing deterministic lookup
    # by table_id (fetching known siblings), not relevance-based retrieval.
    # The "no scroll for retrieval" rule applies to the main search pipeline only.
    sibling_tasks = []
    for tid in table_ids:
        must_conditions = [
            FieldCondition(key="table_id", match=MatchValue(value=tid)),
            FieldCondition(key="is_current_version", match=MatchValue(value=1)),
        ]
        if not include_pii:
            must_conditions.append(FieldCondition(key="contains_pii", match=MatchValue(value=0)))
        sibling_tasks.append(
            client.scroll(
                collection_name=COLLECTION_NAME,
                scroll_filter=Filter(must=must_conditions),
                limit=4,  # max 4 representations per table
                with_payload=True,
            )
        )
    sibling_results = await asyncio.gather(*sibling_tasks)

    # Merge siblings into result set, deduplicating by chunk_id
    existing_ids = {c["chunk_id"] for c in chunks}
    for result_batch, _ in sibling_results:
        for point in result_batch:
            chunk_id = point.payload["chunk_id"]
            if chunk_id not in existing_ids:
                chunks.append(point.payload)
                existing_ids.add(chunk_id)

    return chunks
```

### Parent + Sibling Context Expansion — Full Specification

```python
# src/vector_db/parent_child_retrieval.py

async def expand_context(
    chunks: list[dict],
    include_parents: bool = True,
    include_siblings: bool = True,
    client: AsyncQdrantClient | None = None,
    include_pii: bool = False,
) -> list[dict]:
    """
    Expands reranked chunks with parent context and table siblings.
    
    Parent expansion: For each chunk with a parent_chunk_id, fetches the
    corresponding parent text blob from the manda_parent_chunks collection.
    Parent chunks contain ~2048 tokens of surrounding context, providing
    the synthesizer with enough context to avoid hallucinating connections.
    
    Sibling expansion: For table chunks (is_table=1), fetches all 4
    representation siblings sharing the same table_id. Delegates to
    fetch_table_siblings().
    
    Args:
        chunks: Reranked chunks from retrieval (list of payload dicts).
        include_parents: Whether to fetch parent context (from retrieval config).
        include_siblings: Whether to fetch table siblings (from retrieval config).
        client: AsyncQdrantClient. If None, uses get_qdrant_client().
        include_pii: Compliance authorization override. If False, filters out PII content.
    Returns:
        Expanded list of chunk dicts. Each chunk that has a parent gets a
        "parent_text" key added. Table chunks are augmented with sibling
        representations. Original chunks are always preserved.
    """
    if client is None:
        client = get_qdrant_client()
    
    expanded = [dict(c) for c in chunks]  # copy dicts — prevents mutation of reranked_results
    
    # Parent expansion
    if include_parents:
        parent_ids = list({
            c["parent_chunk_id"] for c in chunks
            if c.get("parent_chunk_id")
        })
        if parent_ids:
            must_conditions = [
                FieldCondition(
                    key="chunk_id",
                    match=MatchAny(any=parent_ids)
                ),
                FieldCondition(
                    key="is_current_version",
                    match=MatchValue(value=1)
                ),
            ]
            if not include_pii:
                must_conditions.append(
                    FieldCondition(key="contains_pii", match=MatchValue(value=0))
                )
            parent_results = await client.scroll(
                collection_name=PARENT_COLLECTION_NAME,
                scroll_filter=Filter(must=must_conditions),
                limit=len(parent_ids),
                with_payload=True,
                with_vectors=False,
            )
            parent_map = {
                p.payload["chunk_id"]: p.payload.get("text", "")
                for p in parent_results[0]
            }
            for chunk in expanded:
                pid = chunk.get("parent_chunk_id")
                if pid and pid in parent_map:
                    chunk["parent_text"] = parent_map[pid]
    
    # Sibling expansion (table representations)
    if include_siblings:
        expanded = await fetch_table_siblings(
            expanded,
            required_representations=["narrative", "row_by_row", "metrics_summary", "markdown"],
            client=client,
            include_pii=include_pii,
        )
    
    return expanded
```

### Agent 5 — Quality Assessor Agent
**Primary: heuristics (no LLM, ~60% of queries) | Fallback: gemini-3.1-flash-lite on ambiguity**
**JSON mode when LLM invoked: response_format={"type": "json_object"}**

Quality thresholds by query type:
```python
THRESHOLDS = {
    "financial":   {"relevance": 0.7, "completeness": 0.7, "precision": 0.8},
    "legal":       {"relevance": 0.8, "completeness": 0.6, "precision": 0.7},
    "summary":     {"relevance": 0.6, "completeness": 0.8, "precision": 0.5},
    "comparative": {"relevance": 0.65, "completeness": 0.7, "precision": 0.6},
    "multi_hop":   {"relevance": 0.65, "completeness": 0.7, "precision": 0.65},
}
```

Heuristic signals (no LLM): mean reranker score of top-5 (relevance), entity coverage fraction (completeness), fiscal-year-matched chunk fraction (precision). LLM invoked only when any score is within ±0.1 of its threshold.

**CRITICAL: `quality_assessor_node` must set `force_refusal` in its return.**
Conditional edge functions must NOT mutate state — they only read state and return routing strings. The quality assessor node sets `force_refusal` based on whether context is sufficient:
```python
async def quality_assessor_node(state: AgentState) -> AgentState:
    """
    Evaluates context quality using heuristic rules (fast, zero LLM cost)
    and falls back to LLM-based evaluation if heuristic scores are close (±0.1)
    to query-type thresholds.
    
    Heuristic rules:
    - Relevance: mean reranker score of the top-5 retrieved chunks.
    - Completeness: fraction of query's extracted entities present in retrieved text.
    - Precision: fraction of chunks matching the target fiscal year (if specified).
    """
    query_type = state["query_type"]
    thresholds = THRESHOLDS.get(query_type, {"relevance": 0.7, "completeness": 0.7, "precision": 0.7})
    
    # 1. Compute Relevance (mean of top-5 reranker scores)
    chunks = state.get("reranked_results", [])
    top_5 = chunks[:5]
    relevance = sum(c.get("reranker_score", 0.0) for c in top_5) / max(len(top_5), 1) if top_5 else 0.0
    
    # 2. Compute Completeness (entity coverage fraction)
    entities = []
    for cat, val in state.get("parsed_intent", {}).get("extracted_entities", {}).items():
        if isinstance(val, list):
            entities.extend([str(v) for v in val if v])
        elif val:
            entities.append(str(val))
            
    if not entities:
        completeness = 1.0
    else:
        context_text = " ".join([c.get("text", "").lower() for c in chunks])
        covered = sum(1 for e in entities if e.lower() in context_text)
        completeness = covered / len(entities)
        
    # 3. Compute Precision (fiscal-year-matched chunk fraction)
    target_year = state.get("extracted_filters", {}).get("fiscal_year")
    if not target_year:
        target_year = state.get("parsed_intent", {}).get("extracted_entities", {}).get("fiscal_years")
        
    target_years = set()
    if isinstance(target_year, str):
        target_years = {target_year}
    elif isinstance(target_year, list):
        target_years = set(str(y) for y in target_year if y)
        
    if not target_years:
        precision = 1.0
    else:
        matched = sum(1 for c in chunks if c.get("fiscal_year") in target_years)
        precision = matched / max(len(chunks), 1) if chunks else 0.0
        
    heuristic_breakdown = {
        "relevance": round(relevance, 2),
        "completeness": round(completeness, 2),
        "precision": round(precision, 2)
    }
    
    # Check if any score is within ±0.1 of its threshold to trigger LLM fallback
    needs_llm = False
    for dim, threshold in thresholds.items():
        if abs(heuristic_breakdown[dim] - threshold) <= 0.1:
            needs_llm = True
            break
            
    if needs_llm:
        # LLM Fallback evaluation
        system_prompt = (
            "You are an expert M&A Diligence Context Evaluator. Assess if the provided context chunks "
            "are sufficient to answer the user's query.\n"
            "Evaluate on three dimensions (score 0.0 to 1.0):\n"
            "1. relevance: Do chunks contain query-relevant clauses/tables?\n"
            "2. completeness: Are all entities/topics in the query covered?\n"
            "3. precision: Are there irrelevant or mismatching document sections?\n"
            "Return a JSON object matching the Quality Assessor output schema."
        )
        context_preview = "\n\n".join([f"Chunk [{c.get('chunk_id')}]: {c.get('text')}" for c in chunks[:10]])
        user_prompt = f"Query: {state['current_query']}\n\nContext:\n{context_preview}"
        
        # Get model via budget tracker
        model = await BudgetTracker.get_instance().get_model_for_agent()
        response = await call_structured_agent(
            system_prompt=system_prompt,
            user_prompt=user_prompt,
            model=model,
            temperature=0.0
        )
        
        quality_breakdown = response.get("quality_breakdown", heuristic_breakdown)
        score = response.get("context_quality_score", round(sum(quality_breakdown.values()) / 3.0, 2))
        missing = response.get("missing_aspects", [])
        method = "llm"
    else:
        # Use heuristic scores
        quality_breakdown = heuristic_breakdown
        score = round(sum(heuristic_breakdown.values()) / 3.0, 2)
        method = "heuristic"
        missing = []
        if quality_breakdown["relevance"] < thresholds["relevance"]:
            missing.append("Relevance score below required threshold.")
        if quality_breakdown["completeness"] < thresholds["completeness"]:
            missing.append("Completeness score below required threshold; missing entities.")
        if quality_breakdown["precision"] < thresholds["precision"]:
            missing.append("Precision score below required threshold; target year mismatch.")
            
    iteration = state["rewrite_iteration"]
    
    # Reuse the existing _meets_type_threshold() by passing a partial state dict
    threshold_state = {
        "quality_breakdown": quality_breakdown,
        "query_type": query_type,
    }
    should_refuse = (
        iteration >= 2 and
        (score < 0.3 or not _meets_type_threshold(threshold_state))
    )
    
    return {
        **state,
        "context_quality_score": score,
        "quality_breakdown": quality_breakdown,
        "quality_method": method,
        "missing_aspects": missing,
        "force_refusal": should_refuse,
    }
```

### Agent 7 — Answer Synthesis Agent
**Model: `get_model_for_synthesis()` → gemini-3.5-flash (budget) / gemini-3.1-flash-lite (fallback)**
**⚠ Verify both model strings against LiteLLM docs**
**Temp: 0.1 | Tokens: 3000 | JSON mode: OFF for this agent (prose answer)**

Citation format includes version and computed flags:

- Hallucination validation failed twice

**CRITICAL: Forced refusal via state flag:**
The `quality_assessor_node` (Agent 5) sets `state["force_refusal"] = True` in its return when context is insufficient after max rewrites. The edge function `route_after_quality_check` only *reads* this flag to decide routing — it never mutates state. The synthesizer node checks the flag at the top:
```python
async def answer_synthesizer_node(state: AgentState) -> AgentState:
    if state.get("force_refusal", False):
        # Generate structured refusal response, NOT an answer.
        return {**state, "generated_answer": _generate_refusal(state),
                "citations": [], "confidence_score": 0.0,
                "validation_status": "passed"}  # skip validation for refusals
    # Normal synthesis path...

def _generate_refusal(state: AgentState) -> str:
    """
    Generates a structured refusal when context is insufficient.
    Does NOT call an LLM — deterministic template to avoid budget consumption
    on a failed query that has already used 2 rewrite iterations.
    
    Args:
        state: AgentState with missing_aspects, query_type, extracted_filters.
    Returns:
        Structured refusal string with actionable information for the analyst.
    """
    missing = state.get("missing_aspects", ["relevant context"])
    query_type = state["query_type"]
    filters = state.get("extracted_filters", {})
    fiscal_year = filters.get("fiscal_year", "unspecified")
    doc_category = filters.get("document_category", "unspecified")
    
    missing_str = ", ".join(missing) if missing else "sufficient relevant context"
    
    return (
        f"**Insufficient Context for {query_type.title()} Query**\n\n"
        f"After {state.get('rewrite_iteration', 0)} retrieval attempts, "
        f"the system could not find adequate context to answer this question reliably.\n\n"
        f"**What is missing:** {missing_str}\n"
        f"**Document categories likely to contain this information:** {doc_category}\n"
        f"**Fiscal year/period needed:** {fiscal_year}\n\n"
        f"**Recommended actions:**\n"
        f"- Verify the relevant documents have been uploaded to the data room\n"
        f"- Try rephrasing the query with more specific terms\n"
        f"- Check if the information is in a document category not yet ingested"
    )
```

## PERFORMANCE TARGETS (Revised)

### Ingestion
- Target: 100 pages/minute
- Embedding batch size: 32 (bge-m3 on GPU via run_in_executor)
- Parallel documents: `asyncio.Semaphore(limit=4)` — prevents VRAM contention
- Qdrant batch upsert: 100 vectors per call

### Query Pipeline
```
Embedding queries (bge-m3, GPU, non-blocking) :  ~60ms
Dense + sparse search in parallel             : ~200ms
flatten_deduplicate + RRF fusion              :  ~30ms
Cross-encoder rerank (bge-reranker, GPU)      : ~280ms
Parent + sibling expansion                    : ~100ms
─────────────────────────────────────────────────────────
Total retrieval pipeline                      : ~670ms

gemini-3.1-flash-lite Agent 1                :  ~600ms
Quality assessor (heuristic path)             :  ~20ms  (no LLM call)

End-to-end simple query (no rewrite)          : ~5–6s

```

## IMPLEMENTATION RULES (Non-Negotiable)

### Code Quality
Every function must have: full type hints (Python 3.10+), docstring with Args/Returns/Raises, structured JSON logging at entry/exit/error, explicit error handling with meaningful messages.

### Qdrant Operations
All Qdrant operations must use `AsyncQdrantClient` (not `QdrantClient`) and be awaited. Include exponential backoff retry (max 3 attempts, base delay 1s, max delay 8s). Handle collection-not-found gracefully. Log operation latency on every call.

### Embedding and Reranking
Never call `SentenceTransformer.encode()` or cross-encoder `predict()` directly in `async` functions. Always wrap in `loop.run_in_executor()` using `asyncio.get_running_loop()` — **NOT** `asyncio.get_event_loop()` which is deprecated in Python 3.10+ and broken in 3.12. Use the module-level `_embed_executor` for embedding and `_rerank_executor` for reranking — these are separate `ThreadPoolExecutor` pools to prevent embed/rerank from starving each other under concurrent load. Also wrap `compute_sparse_bm25()` in `_embed_executor` — it is synchronous CPU computation.

### LangGraph Async Invocation
All LangGraph nodes are `async def`. The compiled graph must be invoked with `await app.ainvoke(state, config=config)` — NOT `app.invoke()`. Using `app.invoke()` with async nodes will deadlock.

### Gemini Model Strings
All model strings (e.g., `"gemini/gemini-3.1-flash-lite"`) must be verified against current LiteLLM Google provider documentation before the config is finalized. Mark all model string constants with a `# ⚠ VERIFY` comment. Incorrect strings cause silent routing failures.

### Financial Data
- Never round or approximate verbatim numbers
- Store `raw_value`, `normalized_value` (raw × scale_factor), `currency`, `scale_factor` in payload
- Compare only `normalized_value` across documents in Financial Verifier
- Distinguish `content_type: "computed_metric"` from `"clean"` in all citations
- Compute all derived metrics (margins, CAGR) in pandas, never via LLM
- Tag every computed figure with `citation_chain` showing its inputs

### LangGraph State
- Use `Annotated[list[dict], operator.add]` for accumulating fields (`rewrite_history`, `agent_trace`)
- Do NOT use lambda reducers — they are not picklable and break PostgresSaver
- Ensure the explicit edge `query_rewriter → parallel_retrieval_executor` is in the graph definition

### Refusal Conditions
The answer synthesizer must return a structured "insufficient context" response (not a hallucinated answer) when:
- Quality score < 0.3 after 2 rewrite attempts
- Hallucination validator returns `status=failed` twice
- The refusal must specify: what information is missing, which document categories would contain it, which fiscal years or time periods are needed

## BUILD ORDER

Implement in this exact sequence:

1. **Token counter** — `AutoTokenizer(bge-m3)` utility, validate against word-split approximation
2. **Qdrant collection setup** — `VECTOR_SIZE=1024` constant locked first; HNSW `m=16, ef_construct=200`; payload indexes using INTEGER not BOOL for boolean fields; quantization with `rescore=True, oversampling=2.0`
3. **Async embedding wrapper** — `embed_texts_async()` via `run_in_executor`, validate non-blocking
4. **FastEmbed BM25** — `compute_sparse_bm25()` with full SparseVector conversion spec
5. **Document processors** — PDF (layout processor + legal clause segmenter + table stitcher), DOCX (clean + redline), PPTX, Excel (normalizer + 4-rep converter)
6. **Version resolver** — version lifecycle, `is_current_version` assignment at ingestion
7. **PII detector** — flag HR/comp documents before indexing
8. **Chunking pipeline** — structural → semantic (max_tokens=800) → financial (pandas metrics)
9. **`flatten_deduplicate` + RRF fusion** — validate pure rank-based output with unit tests
10. **Hybrid search** — dense + sparse with metadata-as-filter; validate QUANTIZED_SEARCH_PARAMS
11. **Async reranker** — `bge-reranker-v2-m3` via `run_in_executor`; validate non-blocking
12. **Parent + sibling retrieval** — table_id-based sibling fetch
13. **Budget tracker + rate limiter** — Postgres-backed, sliding window RPM, verify model strings
14. **LiteLLM wrapper** — `call_structured_agent()` with JSON mode, retry on parse failure
15. **Each agent** — in workflow order; Agent 2 is code only; verify JSON mode on agents 1/5/6/8
16. **LangGraph graph** — `operator.add` reducers, `PostgresSaver` checkpointer, explicit `rewriter→executor` edge, conditional edge functions
17. **Streamlit UI** — live budget display in sidebar, version browser, computed metric flags
18. **FastAPI layer** — immutable audit log on every query endpoint
19. **Docker Compose** — with Ollama model pre-pull entrypoint
20. **Golden Q&A evaluation suite** — 50+ examples; scored recall/precision/faithfulness on schedule
